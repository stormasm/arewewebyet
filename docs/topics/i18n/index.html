Internationalisation

Internationalisation is still a work in progress. There is no standard mature implementation of i18n in Rust, although there are many newer libraries aiming to fill this gap.

  • fluent

    A localization system designed to unleash the entire expressive power of natural language translations.

  • gettext-rs

    GNU Gettext FFI binding for Rust

  • intl_pluralrules

    Unicode Plural Rules categorizer for numeric input.

  • num-format

    A Rust crate for producing string-representations of numbers, formatted according to international standards

  • rust_icu

    Native bindings to the ICU4C library from Unicode.

  • unic

    UNIC: Unicode and Internationalization Crates

Upcoming

We are waiting for the following to reach maturity to be released on crates.io:
  • ICU4X

    Solving i18n for client-side and resource-constrained environments
